# 岸坡多指标风险评估模型系统 - 快速上手指南

## 一、系统概述

本系统包含**10个模型**，分为**3大类指标**和**1个综合风险评估模型**，用于评估河岸/岸坡稳定性风险。

### 模型分类
```
┌─────────────────────────────────────────────────────────────┐
│                     综合风险评估 (Risk Level)                  │
├─────────────────┬─────────────────┬─────────────────────────┤
│   水流动力指标   │   河床演变指标   │      地质工程指标        │
│    (wNM权重)    │    (wRE权重)    │       (wGE权重)         │
├─────────────────┼─────────────────┼─────────────────────────┤
│ 1. 抗冲击流速    │ 4. 坡度 (Sa)    │ 7. 土壤成分 (Dsed)       │
│    (Ky)         │ 5. 高差 (Zb)    │ 8. 岸坡防护 (PL)         │
│ 2. 流量当量      │ 6. 近岸冲刷(Ln)  │ 9. 荷载控制 (LC)         │
│    (PQ)         │                 │                         │
│ 3. 水位波动      │                 │                         │
│    (Zd)         │                 │                         │
└─────────────────┴─────────────────┴─────────────────────────┘
```

## 三、各模型详细参数说明

### 3.1 地质工程类指标

#### 1. 土壤成分模型 (Soil Composition / Dsed)
**文件名**: `soilComposition.py`

**输入参数**:
```json
{
  "hs": 0.5,              // 砂土层厚度 (米)
  "hc": 2.0,              // 土层总覆盖厚度 (米)
  "risk-threshold": [0.3, 0.5, 0.7]  // 风险阈值 [低,中,高] (可选，默认从模板读取)
}
```

**输出结果**:
```json
{
  "case-id": "模型实例ID",
  "Dsed": 0.25,           // 土壤成分指标 = hs/hc
  "risk-level": [0,0,1,0] // 风险等级 [I,II,III,IV]
}
```

**风险等级判断**:
- Dsed < 0.3 → 风险等级 I (安全) [1,0,0,0]
- Dsed < 0.5 → 风险等级 II (轻度) [0,1,0,0]
- Dsed < 0.7 → 风险等级 III (中度) [0,0,1,0]
- Dsed ≥ 0.7 → 风险等级 IV (重度) [0,0,0,1]

---

**文件名**: `slopeProtection.py`

**输入参数**:
```json

  "protection-level": "systemic"  // 防护等级: "systemic"(系统)/"normal"(常规)/"low"(低)/"no"(无)
```

**输出结果**:

### 3.2 河床演变类指标

#### 4. 高差模型 (Height Difference / Zb)
**文件名**: `heightDifference.py`

**输入参数**:
```json
{
  "dem-id": "数字高程模型ID",      // DEM数据源
  "section-geometry": {            // 剖面几何信息
    "type": "LineString",
    "coordinates": [[x1,y1], [x2,y2]]
  },
  "risk-threshold": [2.0, 4.0, 6.0] // 风险阈值 (可选)
}
```

**输出结果**:
```json
{
  "case-id": "模型实例ID",
  "Zb": 3.5,              // 高差值 (米)
  "risk-level": [0,1,0,0] // 风险等级
}
```

**计算逻辑**: Zb = -points[deepest_index][2] (最深点高程的负值)

**风险等级判断**:
- Zb < 2.0 → I级 [1,0,0,0]
- Zb < 4.0 → II级 [0,1,0,0]
- Zb < 6.0 → III级 [0,0,1,0]
- Zb ≥ 6.0 → IV级 [0,0,0,1]

#### 5. 坡度模型 (Slope Rate / Sa)
**文件名**: `slopeRate.py`

**输入参数**:
```json

  "section-geometry": {
    "type": "LineString",
    "coordinates": [[x1,y1], [x2,y2]]
```

**计算逻辑**: Sa = -min(Sa_v) (最小坡度值的负值)

**风险等级判断**:
- Sa < 15° → I级
- Sa < 25° → II级
- Sa < 35° → III级
- Sa ≥ 35° → IV级

#### 6. 近岸冲刷模型 (Nearshore Flush / Ln)
**文件名**: `nearshoreFlush.py`

**输入参数**:
```json
{
  "bench-id": "基准DEM数据ID",      // 当前时相DEM
  "ref-id": "参考DEM数据ID",        // 对比时相DEM
  "current-timepoint": "2024-01-15", // 当前时间点
  "comparison-timepoint": "2020-01-15", // 对比时间点
  "section-geometry": {
    "type": "LineString",
    "coordinates": [[x1,y1], [x2,y2]]
  },
  "risk-threshold": [0.5, 1.0, 1.5]  // 冲刷速率阈值 (米/月)
}
```

**输出结果**:
```json
{
  "case-id": "模型实例ID",
  "Ln": 0.8,              // 近岸冲刷速率 (米/月)
  "risk-level": [0,0,1,0]
}
```

**计算逻辑**: Ln = -min_z / time (最深点高程变化/时间差，单位：月)

**执行流程**:
1. 执行Region Flush模型生成冲刷区域tif文件
2. 基于冲刷结果执行Section View剖面分析
3. 计算Ln指标

#### 8. 抗冲击流速模型 (Anti-Impact Speed / Ky)
**文件名**: `antiImpactSpeed.py`

**输入参数**:
```json
{
  "dem-id": "数字高程模型ID",
  "section-geometry": {
    "type": "LineString",
    "coordinates": [[x1,y1], [x2,y2]]
  },
  "segment": "Mzs",       // 河段
  "year": "2024",         // 年份
  "set": "standard",      // 数据集名称
  "water-qs": "45000",    // 流量值 (m³/s)
  "tidal-level": "zc",    // 潮位等级: "xc"(小潮)/"zc"(中潮)/"dc"(大潮) 或数值自动映射
  "risk-threshold": [0.5, 0.3, 0.1]  // Ky阈值
}
```

**输出结果**:
```json
{
  "case-id": "模型实例ID",
  "Ky": 0.45,             // 抗冲击流速指标
  "risk-level": [0,0,1,0]
}
```

**计算逻辑**: 
- 获取坡脚坐标(slope_foot)
- 在水动力场中找到最近点
- 提取26个时间点的流速数据
- Ky = 2.4231 × exp(-0.547 × max_velocity)

**执行流程**:
1. 执行Section View获取坡脚位置
2. 执行Hydrodynamic获取流场数据
3. 计算Ky指标

**风险等级判断**:
- Ky > 0.5 → I级
- Ky > 0.3 → II级
- Ky > 0.1 → III级
- Ky ≤ 0.1 → IV级

---

#### 9. 水位波动模型 (Water-Level Fluctuation / Zd)
**文件名**: `waterLevelFluctuation.py`

**输入参数**:
```json
{
  "dem-id": "数字高程模型ID",
  "section-geometry": {
    "type": "LineString",
    "coordinates": [[x1,y1], [x2,y2]]
  },
  "segment": "Mzs",
  "year": "2024",
  "set": "standard",
  "water-qs": "45000",
  "tidal-level": "zc",
  "risk-threshold": [0.1, 0.2, 0.3]
}
```

**输出结果**:
```json
{
  "case-id": "模型实例ID",
  "Zd": 0.15,             // 水位波动指标
  "risk-level": [0,0,1,0]
}
```

**计算逻辑**:
- 从水动力数据中提取26个时间点的水位序列p
- 识别波峰Zmax1、Zmax2和波谷Zmin1、Zmin2
- A = 0.5 × (Zmax1 + Zmax2 - Zmin1 - Zmin2)
- B = 根据流量插值获取的系数
- Zd = B + A/Z (Z为基准水位)

**执行流程**:
1. 执行Section View获取坡脚位置
2. 执行Hydrodynamic获取水位数据
3. 计算Zd指标

### 3.4 综合风险评估模型

#### 10. 风险等级综合评估模型 (Risk Level)
**文件名**: `riskLevel.py`

**输入参数**:
```json
{
  // ===== 基础输入参数 =====
  "bench-id": "基准DEM数据ID",
  "ref-id": "参考DEM数据ID",
  "current-timepoint": "2024-01-15",
  "comparison-timepoint": "2020-01-15",
  "section-geometry": {
    "type": "LineString",
    "coordinates": [[x1,y1], [x2,y2]]
  },
  
  // ===== 水动力参数 =====
  "segment": "Mzs",
  "set": "standard",
  "water-qs": "45000",
  "tidal-level": "zc",
  
  // ===== 地质工程参数 =====
  "hs": 0.5,              // 砂土层厚度
  "hc": 2.0,              // 覆盖层厚度
  "protection-level": "systemic",  // 防护等级
  "control-level": "strict",       // 控制等级
  
  // ===== 风险阈值配置 (可选) =====
  "risk-thresholds": {
    "Dsed": [0.3, 0.5, 0.7],    // 土壤成分
    "Zb": [2.0, 4.0, 6.0],      // 高差
    "Sa": [15, 25, 35],         // 坡度
    "Ln": [0.5, 1.0, 1.5],      // 近岸冲刷
    "PQ": [1000, 2000, 3000],   // 流量当量
    "Ky": [0.5, 0.3, 0.1],      // 抗冲击流速
    "Zd": [0.1, 0.2, 0.3],      // 水位波动
    "all": [0.2, 0.4, 0.6]      // 综合风险阈值
  },
  
  // ===== 权重配置 (可选，默认从模板读取) =====
  "wRE": [0.3, 0.4, 0.3],     // 河床演变类指标权重 [坡度,高差,近岸冲刷]
  "wNM": [0.4, 0.3, 0.3],     // 水流动力类指标权重 [抗冲击流速,流量当量,水位波动]
  "wGE": [0.4, 0.4, 0.2],      // 地质工程类指标权重 [土壤成分,岸坡防护,荷载控制]
  "wRL": [0.3, 0.3, 0.4]      // 三大类指标权重 [水流动力,河床演变,地质工程]
}
```

**输出结果**:
```json
{
  "case-id": "模型实例ID",
  "multi-indicator-ids": {
    "Dsed": "土壤成分模型ID",
    "PL": "岸坡防护模型ID",
    "LC": "荷载控制模型ID",
    "Zb": "高差模型ID",
    "Sa": "坡度模型ID",
    "Ln": "近岸冲刷模型ID",
    "PQ": "流量当量模型ID",
    "Ky": "抗冲击流速模型ID",
    "Zd": "水位波动模型ID"
  },
  "result": 0.35,         // 综合风险值 (0-1之间)
  "risk-level": [0,1,0,0] // 最终风险等级
}
```

**权重计算逻辑**:

```
水流动力矩阵 = [Ky风险, PQ风险, Zd风险] × wNM权重
河床演变矩阵 = [Sa风险, Zb风险, Ln风险] × wRE权重  
地质工程矩阵 = [Dsed风险, PL风险, LC风险] × wGE权重

综合矩阵 = [水流动力结果, 河床演变结果, 地质工程结果] × wRL权重

风险值 = 综合矩阵的第3项 + 第4项
```

**执行流程**:
1. 并行启动9个子模型
2. 等待所有子模型完成
3. 读取各子模型的风险等级矩阵
4. 按权重计算综合风险值
5. 输出最终风险等级

## 五、注意事项

### 5.1 潮位等级映射规则

`tidal-level` 参数可以是：
- **字符串值**: "xc"(小潮), "zc"(中潮), "dc"(大潮)
- **数值**: 自动映射
  - < 1.5 → "xc"
  - < 2.3 → "zc"
  - ≥ 2.3 → "dc"

### 5.2 流量值映射规则

输入的 `water-qs` 会自动映射到最近的可用节点：
- 系统会查找水文数据目录下的所有可用流量节点
- 选择最接近输入值的节点进行计算

### 5.3 风险阈值说明

- 所有模型的 `risk-threshold` 参数都是**可选的**
- 传入 "NONE" 或省略时，系统会从 `DIR_RESOURCE_RISKLEVEL_THRESHOLD_TEMPLATE` 读取默认值
- 风险等级矩阵格式: `[I级, II级, III级, IV级]` (one-hot编码)

### 5.4 模型API常量

```python
# 地质工程类
config.API_MI_SOIL_COMPOSITION      # 土壤成分
config.API_MI_SLOPE_PROTECTION      # 岸坡防护
config.API_MI_LOAD_CONTROL          # 荷载控制

# 河床演变类
config.API_MI_HEIGHT_DIFFERENCE     # 高差
config.API_MI_SLOPE_RATE            # 坡度
config.API_MI_NEARSHORE_FLUSH       # 近岸冲刷

# 水流动力类
config.API_MI_FLOW_EQUIVALENT       # 流量当量
config.API_MI_ANTI_IMPACT_SPEED     # 抗冲击流速
config.API_MI_WATER_LEVEL_FLUCTUATION  # 水位波动

# 综合评估
config.API_MI_RISK_LEVEL            # 风险等级

# 依赖的基础模型
config.API_RE_SECTION_VIEW          # 剖面分析
config.API_RE_REGION_FLUSH          # 区域冲刷
config.API_NM_HYDRODYNAMIC          # 水动力
```
