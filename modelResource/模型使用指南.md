# 岸坡多指标风险评估模型系统 - 快速上手指南

## 一、系统概述

本系统包含**10个多指标风险评估模型**和**3个基础支撑模型**，分为**3大类指标**和**1个综合风险评估模型**，用于评估河岸/岸坡稳定性风险。

### 模型分类
```
┌─────────────────────────────────────────────────────────────────────────┐
│                          综合风险评估 (Risk Level)                          │
├──────────────────────┬──────────────────────┬───────────────────────────┤
│    水流动力指标       │    河床演变指标       │      地质工程指标          │
│     (wNM权重)        │     (wRE权重)        │       (wGE权重)           │
├──────────────────────┼──────────────────────┼───────────────────────────┤
│ 1. 抗冲击流速 (Ky)   │ 4. 坡度 (Sa)         │ 7. 土壤成分 (Dsed)        │
│ 2. 流量当量 (PQ)     │ 5. 高差 (Zb)         │ 8. 岸坡防护 (PL)          │
│ 3. 水位波动 (Zd)     │ 6. 近岸冲刷 (Ln)      │ 9. 荷载控制 (LC)          │
└──────────────────────┴──────────────────────┴───────────────────────────┘
         │                       │                       │

```

## 三、基础支撑模型详解

### 3.1 河床演变类基础模型

#### 3.1.1 剖面分析模型 (Section View)
**文件名**: `riverbedEvolution/sectionView.py`

**功能说明**: 基于DEM数据和剖面几何信息，提取岸坡剖面的高程数据，计算坡度、高差等几何参数。

**输入参数**:
```json
{
  "dem-id": "数字高程模型ID",
  "section-geometry": {
    "type": "LineString",
    "coordinates": [[x1,y1], [x2,y2]]
  }
}
```

**输出结果**:
```json
{
  "case-id": "模型实例ID",
  "visualization-txt": "section.txt",
  "raw-json": "section.json",
  "interval": 采样间隔
}
```

**section.json 包含内容**:
- `points`: 剖面上100个采样点的三维坐标 `[x, y, z]`
- `step`: 采样间隔（米）
- `Sa_h`: 水平方向坡度数组
- `points_v`: 垂直方向采样点
- `Sa_v`: 垂直方向坡度数组
- `deepest_index`: 最深点索引
- `slope_foot_index`: 坡脚点索引
- `points_er_verified`: 侵蚀模型验证点
- `points_er`: 侵蚀模型点

**被依赖的模型**: 高差模型、坡度模型、近岸冲刷模型、抗冲击流速模型、水位波动模型

---

#### 3.1.2 区域冲刷模型 (Region Flush)
**文件名**: `riverbedEvolution/regionFlush.py`

**功能说明**: 对比两个时相的DEM数据，计算河床冲淤变化，生成冲淤分布图。

**输入参数**:
```json
{
  "bench-id": "基准DEM数据ID",      // 当前时相DEM
  "ref-id": "参考DEM数据ID",        // 对比时相DEM
  "region-geometry": "NONE" 或 GeoJSON // 可选：裁剪区域
}
```

**执行流程**:
1. **Global Flush**: 先计算全江域的冲淤变化
2. **Region Flush**（可选）: 如果指定了区域几何，则对全江结果进行裁剪

**输出结果**:
```json
{
  "case-id": "模型实例ID",
  "raw-tif": "flush.tif",         // 冲淤栅格数据
  "extent-json": "flush.json",     // 范围坐标
  "visualization-png": "flush.png" // 彩色可视化图
}
```

**被依赖的模型**: 近岸冲刷模型

---

### 3.2 数值模型类基础模型

#### 3.2.1 水动力模型 (Hydrodynamic)
**文件名**: `numericalModel/hydrodynamic.py`

**功能说明**: 提供已计算好的水动力场数据，包括流速、水位等信息。

**输入参数**:
```json
{
  "segment": "Mzs",       // 河段名称
  "year": "2024",         // 年份
  "set": "standard",      // 数据集名称
  "water-qs": "45000",    // 流量值 (m³/s)
  "tidal-level": "zc"     // 潮位等级: "xc"(小潮)/"zc"(中潮)/"dc"(大潮)
}
```

**参数映射逻辑**:
- **潮位映射**: 数值自动映射
  - < 1.5 → "xc" (小潮)
  - < 2.3 → "zc" (中潮)
  - ≥ 2.3 → "dc" (大潮)
- **流量映射**: 自动匹配最近的可用流量节点

**输出结果**:
```json
{
  "case-id": "模型实例ID",
  "raw-txts": ["hydrodynamic/.../0.txt", ...], // 26个时间点的原始数据
  "visualization-uv-bin": ["hydrodynamic/.../uv_0.bin", ...], // 流速二进制文件
  "visualization-station-bin": "hydrodynamic/.../station.bin",
  "visualization-description-json": "hydrodynamic/.../flow_field_description.json"
}
```

**数据目录结构**:
```
{segment}/{year}/{set}/{water-qs}{tidal-level}/
├── raw/                    # 原始数据
│   ├── 0.txt ~ 25.txt      # 26个时间点的数据
└── renderResource/         # 渲染资源
    ├── uv_0.bin ~ uv_25.bin # 流速数据
    ├── station.bin
    └── flow_field_description.json
```

**被依赖的模型**: 抗冲击流速模型、水位波动模型

## 五、各模型详细参数说明

### 5.1 地质工程类指标

#### 1. 土壤成分模型 (Soil Composition / Dsed)
**文件名**: `soilComposition.py`

**输入参数**:
```json
{
  "hs": 0.5,              // 砂土层厚度 (米)
  "hc": 2.0,              // 土层总覆盖厚度 (米)
  "risk-threshold": [0.3, 0.5, 0.7]  // 风险阈值 [低,中,高] (可选，默认从模板读取)
}
```

**输出结果**:
```json
{
  "case-id": "模型实例ID",
  "Dsed": 0.25,           // 土壤成分指标 = hs/hc
  "risk-level": [0,0,1,0] // 风险等级 [I,II,III,IV]
}
```

**风险等级判断**:
- Dsed < 0.3 → 风险等级 I (安全) [1,0,0,0]
- Dsed < 0.5 → 风险等级 II (轻度) [0,1,0,0]
- Dsed < 0.7 → 风险等级 III (中度) [0,0,1,0]
- Dsed ≥ 0.7 → 风险等级 IV (重度) [0,0,0,1]

---

### 5.2 河床演变类指标

#### 4. 高差模型 (Height Difference / Zb)
**文件名**: `heightDifference.py`

**输入参数**:
```json
{
  "dem-id": "数字高程模型ID",      // DEM数据源
  "section-geometry": {            // 剖面几何信息
    "type": "LineString",
    "coordinates": [[x1,y1], [x2,y2]]
```

**计算逻辑**: Zb = -points[deepest_index][2] (最深点高程的负值)

**风险等级判断**:
- Zb < 2.0 → I级 [1,0,0,0]
- Zb < 4.0 → II级 [0,1,0,0]
- Zb < 6.0 → III级 [0,0,1,0]
- Zb ≥ 6.0 → IV级 [0,0,0,1]

#### 8. 抗冲击流速模型 (Anti-Impact Speed / Ky)
**文件名**: `antiImpactSpeed.py`

**输入参数**:
```json
{
  "dem-id": "数字高程模型ID",
  "section-geometry": {
    "type": "LineString",
    "coordinates": [[x1,y1], [x2,y2]]
  },
  "segment": "Mzs",       // 河段
  "year": "2024",         // 年份
  "set": "standard",      // 数据集名称
  "water-qs": "45000",    // 流量值 (m³/s)
  "tidal-level": "zc",    // 潮位等级: "xc"(小潮)/"zc"(中潮)/"dc"(大潮) 或数值自动映射
  "risk-threshold": [0.5, 0.3, 0.1]  // Ky阈值
}
```

**输出结果**:
```json
{
  "case-id": "模型实例ID",
  "Ky": 0.45,             // 抗冲击流速指标
  "risk-level": [0,0,1,0]
}
```

**计算逻辑**: 
- 获取坡脚坐标(slope_foot)
- 在水动力场中找到最近点
- 提取26个时间点的流速数据
- Ky = 2.4231 × exp(-0.547 × max_velocity)

**执行流程**:
1. 执行Section View获取坡脚位置
2. 执行Hydrodynamic获取流场数据
3. 计算Ky指标

**风险等级判断**:
- Ky > 0.5 → I级
- Ky > 0.3 → II级
- Ky > 0.1 → III级
- Ky ≤ 0.1 → IV级

#### 9. 水位波动模型 (Water-Level Fluctuation / Zd)
**文件名**: `waterLevelFluctuation.py`

**输入参数**:
```json
{
  "dem-id": "数字高程模型ID",
  "section-geometry": {
    "type": "LineString",
    "coordinates": [[x1,y1], [x2,y2]]
  },
  "segment": "Mzs",
  "year": "2024",
  "set": "standard",
  "water-qs": "45000",
  "tidal-level": "zc",
  "risk-threshold": [0.1, 0.2, 0.3]
}
```

**输出结果**:
```json
{
  "case-id": "模型实例ID",
  "Zd": 0.15,             // 水位波动指标
  "risk-level": [0,0,1,0]
}
```

**计算逻辑**:
- 从水动力数据中提取26个时间点的水位序列p
- 识别波峰Zmax1、Zmax2和波谷Zmin1、Zmin2
- A = 0.5 × (Zmax1 + Zmax2 - Zmin1 - Zmin2)
- B = 根据流量插值获取的系数
- Zd = B + A/Z (Z为基准水位)

**执行流程**:
1. 执行Section View获取坡脚位置
2. 执行Hydrodynamic获取水位数据
3. 计算Zd指标

#### 10. 风险等级综合评估模型 (Risk Level)
**文件名**: `riskLevel.py`

```json
  "tidal-level": "zc",

  "hs": 0.5,              // 砂土层厚度
  "hc": 2.0,              // 覆盖层厚度
  "protection-level": "systemic",  // 防护等级
  "control-level": "strict",       // 控制等级
  
  // ===== 风险阈值配置 (可选) =====
  "risk-thresholds": {
    "Dsed": [0.3, 0.5, 0.7],    // 土壤成分
    "Zb": [2.0, 4.0, 6.0],      // 高差
    "Sa": [15, 25, 35],         // 坡度
    "Ln": [0.5, 1.0, 1.5],      // 近岸冲刷
    "PQ": [1000, 2000, 3000],   // 流量当量
    "Ky": [0.5, 0.3, 0.1],      // 抗冲击流速
    "Zd": [0.1, 0.2, 0.3],      // 水位波动
    "all": [0.2, 0.4, 0.6]      // 综合风险阈值
  },
  
  // ===== 权重配置 (可选，默认从模板读取) =====
  "wRE": [0.3, 0.4, 0.3],     // 河床演变类指标权重 [坡度,高差,近岸冲刷]
  "wNM": [0.4, 0.3, 0.3],     // 水流动力类指标权重 [抗冲击流速,流量当量,水位波动]
  "wGE": [0.4, 0.4, 0.2],      // 地质工程类指标权重 [土壤成分,岸坡防护,荷载控制]
  "wRL": [0.3, 0.3, 0.4]      // 三大类指标权重 [水流动力,河床演变,地质工程]
}
```

**输出结果**:
```json
{
  "case-id": "模型实例ID",
  "multi-indicator-ids": {
    "Dsed": "土壤成分模型ID",
    "PL": "岸坡防护模型ID",
    "LC": "荷载控制模型ID",
    "Zb": "高差模型ID",
    "Sa": "坡度模型ID",
    "Ln": "近岸冲刷模型ID",
    "PQ": "流量当量模型ID",
    "Ky": "抗冲击流速模型ID",
    "Zd": "水位波动模型ID"
  },
  "result": 0.35,         // 综合风险值 (0-1之间)
  "risk-level": [0,1,0,0] // 最终风险等级
}
```

**权重计算逻辑**:

```
水流动力矩阵 = [Ky风险, PQ风险, Zd风险] × wNM权重
河床演变矩阵 = [Sa风险, Zb风险, Ln风险] × wRE权重  
地质工程矩阵 = [Dsed风险, PL风险, LC风险] × wGE权重

综合矩阵 = [水流动力结果, 河床演变结果, 地质工程结果] × wRL权重

风险值 = 综合矩阵的第3项 + 第4项
```

**执行流程**:
1. 并行启动9个子模型
2. 等待所有子模型完成
3. 读取各子模型的风险等级矩阵
4. 按权重计算综合风险值
5. 输出最终风险等级

## 七、注意事项

### 7.1 潮位等级映射规则

`tidal-level` 参数可以是：
- **字符串值**: "xc"(小潮), "zc"(中潮), "dc"(大潮)
- **数值**: 自动映射
  - < 1.5 → "xc"
  - < 2.3 → "zc"
  - ≥ 2.3 → "dc"

### 7.2 流量值映射规则

输入的 `water-qs` 会自动映射到最近的可用节点：
- 系统会查找水文数据目录下的所有可用流量节点
- 选择最接近输入值的节点进行计算

### 7.3 风险阈值说明

- 所有模型的 `risk-threshold` 参数都是**可选的**
- 传入 "NONE" 或省略时，系统会从 `DIR_RESOURCE_RISKLEVEL_THRESHOLD_TEMPLATE` 读取默认值
- 风险等级矩阵格式: `[I级, II级, III级, IV级]` (one-hot编码)

### 7.4 模型API常量

```python
# 地质工程类
config.API_MI_SOIL_COMPOSITION      # 土壤成分
config.API_MI_SLOPE_PROTECTION      # 岸坡防护

# 河床演变类
config.API_MI_HEIGHT_DIFFERENCE     # 高差
config.API_MI_SLOPE_RATE            # 坡度
config.API_MI_NEARSHORE_FLUSH       # 近岸冲刷

# 水流动力类
config.API_MI_FLOW_EQUIVALENT       # 流量当量
config.API_MI_ANTI_IMPACT_SPEED     # 抗冲击流速
config.API_MI_WATER_LEVEL_FLUCTUATION  # 水位波动

# 综合评估
config.API_MI_RISK_LEVEL            # 风险等级

# 依赖的基础模型
config.API_RE_SECTION_VIEW          # 剖面分析
config.API_RE_REGION_FLUSH          # 区域冲刷
config.API_NM_HYDRODYNAMIC          # 水动力
```

#### 10.4.2 参数值参考

**protection-level 取值**:

- `"zc"` - 中潮
- `"dc"` - 大潮
